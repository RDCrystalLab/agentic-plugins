# 圖表渲染後審查清單

渲染 PNG 並檢視後，逐項檢查以下項目。每個項目標記通過或需修正。

## 設計審查

- [ ] **視覺結構**：是否符合規劃的概念結構？
- [ ] **視覺模式**：每個區段是否使用了預期的模式？
- [ ] **視線流向**：視線是否按設計順序流過圖表？
- [ ] **視覺層級**：主角元素佔優勢、輔助元素較小？
- [ ] **證據工件**（技術圖表）：是否可讀且正確放置？

## 缺陷檢查

- [ ] **文字裁切**：所有文字都在容器邊界內？沒有溢出？
- [ ] **元素重疊**：形狀和文字沒有意外重疊？
- [ ] **箭頭路徑**：箭頭繞過元素而非穿越？
- [ ] **標籤位置**：標籤位置明確，不會造成歧義？
- [ ] **間距均勻**：相似元素有一致的間距？
- [ ] **文字可讀性**：文字在渲染尺寸下清晰可辨？
- [ ] **構圖平衡**：沒有大面積空洞或過度擁擠的區域？

## 技術正確性

- [ ] **ID 唯一**：所有元素 ID 唯一且為描述性字串？
- [ ] **binding 正確**：箭頭的 `startBinding` / `endBinding` 指向正確元素？
- [ ] **boundElements 同步**：被箭頭連接的形狀已更新其 `boundElements`？
- [ ] **containerId 一致**：容器內文字的 `containerId` 指向正確的父形狀？

## 修正指引

若發現問題，常見修正方式：
- **文字溢出** → 擴大容器的 `width` / `height`，或縮短文字
- **間距不均** → 調整 `x` / `y` 座標，對齊到 10 的倍數
- **箭頭穿越** → 在箭頭的 `points` 陣列中加入中繼點繞道
- **視覺重量失衡** → 調整元素尺寸重新平衡

修正後重新渲染，再次檢查此清單，直到所有項目通過。

# 圖表需求澄清模板

當使用者的描述模糊或不夠具體時，使用以下結構化問題引導澄清。
不需要問所有問題 — 根據情境挑選最相關的即可。

## 受眾與目的

- 這張圖表的**受眾**是誰？（技術人員 / 非技術人員 / 混合）
- 用途是什麼？（簡報 / 文件 / 教學 / 個人筆記）
- 需要多少**細節深度**？（概念性總覽 / 具體技術細節）

## 範圍與邊界

- 要涵蓋哪些**主要元件或概念**？
- 有什麼**不需要**納入的？
- 需要展示資料流嗎？還是只展示結構關係？

## 技術細節（若為技術圖表）

- 涉及哪些**技術棧**？（語言、框架、服務、資料庫）
- 需要展示**真實的 API 端點、事件名稱或資料格式**嗎？
- 有沒有參考文件或 spec 可以查閱？

## 風格偏好

- 偏好哪種**視覺風格**？（乾淨俐落 / 手繪風格）
- 有沒有**現有的圖表**作為參考或要更新的？
- 對色彩或佈局有特殊要求嗎？

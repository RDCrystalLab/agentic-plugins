# 大型 / 多區段圖表策略

**全面性或技術性圖表必須逐段建構 JSON。** 不要嘗試在一次回應中生成整個檔案 — 會導致截斷和品質下降。

## 逐段工作流程

### 階段 1：逐段建構

1. **建立基礎檔案** — 包含 JSON 外殼（`type`、`version`、`appState`、`files`）和第一段元素。
2. **每次編輯加入一個區段。** 每個區段有自己的專屬步驟。
3. **使用描述性字串 ID**（例如 `"trigger_rect"`、`"arrow_fan_left"`）。
4. **按區段命名 seed**（例如區段 1 使用 100xxx，區段 2 使用 200xxx）。
5. **即時更新跨區段 binding** — 當新區段的箭頭連接到先前區段的元素時，同時更新該元素的 `boundElements`。

### 階段 2：整體審查

所有區段就位後，通讀完整 JSON 並檢查：
- 跨區段箭頭的兩端 binding 正確嗎？
- 整體間距平衡嗎？
- 所有 ID 和 binding 都指向實際存在的元素嗎？

### 階段 3：渲染與驗證

執行 SKILL.md 中渲染與驗證段落的 render-view-fix 迴圈。

## 禁止事項

- **不要在一次回應中生成整個圖表。** 你會撞到 token 上限並產生截斷的 JSON。
- **不要寫 Python 生成腳本。** 使用描述性 ID 的手工 JSON 更容易維護。

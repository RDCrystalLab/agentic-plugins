# 技術 / 全面性圖表

當需要繪製真實系統、協定或架構，且受眾需要具體細節時，使用此 cookbook。

## 適用時機

- 繪製真實系統、協定或架構
- 圖表用於教學或解釋
- 受眾需要了解事物的真實樣貌
- 展示多個技術如何整合

## 先研究再動手

**在繪製任何技術內容之前，先研究實際規格。**

如果要繪製協定、API 或框架：
1. 查詢實際的 JSON / 資料格式
2. 找到真實的事件名稱、方法名稱或 API 端點
3. 了解各部件如何實際連接
4. 使用真實術語，而非泛用佔位符

壞例子："Protocol" -> "Frontend"
好例子："AG-UI streams events (RUN_STARTED, STATE_DELTA)" -> "CopilotKit renders via createA2UIMessageRenderer()"

## 證據工件（Evidence Artifacts）

證據工件是具體範例，用以證明圖表的準確性並幫助觀者學習。技術性圖表應包含這些。

| 工件類型 | 使用時機 | 渲染方式 |
|----------|----------|----------|
| **程式碼片段** | API、整合、實作細節 | 深色矩形 + 語法高亮文字（見 color palette） |
| **資料 / JSON 範例** | 資料格式、schema、payload | 深色矩形 + 彩色文字 |
| **事件 / 步驟序列** | 協定、工作流程 | 時間軸模式（線 + 圓點 + 標籤） |
| **UI 模擬** | 展示實際輸出 | 巢狀矩形模擬真實 UI |
| **API / 方法名稱** | 真實的函式呼叫、端點 | 使用文件中的實際名稱，非佔位符 |

**關鍵原則：展示事物的真實樣貌**，而非僅標註名稱。

## 多層縮放架構（Multi-Zoom）

全面性圖表同時在多個縮放層級運作。

### 第 1 層：摘要流程
簡化的總覽，一目了然展示完整管線。通常放在圖表的頂部或底部。

### 第 2 層：區段邊界
標記區域，將相關元件分組。建立視覺「房間」。

### 第 3 層：區段內的細節
證據工件、程式碼片段和具體範例。教育價值的核心所在。

**全面性圖表應包含所有三個層級。**

## 設計流程

### 步驟 0：研究
研究實際規格、格式、事件名稱。見上方「先研究再動手」段落。

### 步驟 1：深入理解
對每個概念問：
- 這個概念**做什麼**？（不是它**是什麼**）
- 概念之間有什麼關係？
- 核心的轉換或流程是什麼？
- **讀者需要看到什麼才能理解？**

### 步驟 2：將概念映射到視覺模式
為每個概念找到映射其行為的視覺模式。讀取 `cookbook/visual-patterns.md` 取得模式庫。

### 步驟 3：確保多樣性
多概念圖表：**每個主要概念必須使用不同的視覺模式**。禁止千篇一律的卡片方格。

### 步驟 4：勾勒流程
在生成 JSON 之前，在腦中描繪眼睛如何穿越圖表。應有清晰的視覺故事。

### 步驟 5：生成 JSON
建立 Excalidraw 元素。讀取 `references/element-templates.md` 取得 JSON 模板，讀取 `references/color-palette.md` 取得色彩。**若圖表較大（>15 個元素），讀取 `cookbook/large-diagram-strategy.md`。**

### 步驟 6：渲染與驗證
生成 JSON 後，必須執行 render-view-fix 迴圈直到圖表正確（見 SKILL.md 的渲染與驗證段落）。

## 品質清單

### 深度與佐證
1. **研究完成**：是否查詢了實際規格、格式、事件名稱？
2. **證據工件**：是否包含程式碼片段、JSON 範例或真實資料？
3. **多層縮放**：是否有摘要流程 + 區段邊界 + 細節？
4. **具體優於抽象**：展示真實內容，而非僅有標籤方框？

### 概念性
5. **同構性**：每個視覺結構是否映射其概念的行為？
6. **論述性**：圖表是否展示了文字無法表達的東西？
7. **多樣性**：每個主要概念是否使用不同的視覺模式？
8. **無千篇一律容器**：避免了卡片方格和等大方框？

### 容器紀律
9. **最少容器**：是否有框的元素可以改為浮動文字？
10. **線條作為結構**：樹狀/時間軸模式是否使用線條+文字而非方框？
11. **字體層級**：字體大小和色彩是否建立了視覺層級？

### 結構性
12. **連線完整**：每個關係都有箭頭或線條？
13. **流向清晰**：視線有清晰的路徑可循？
14. **層級分明**：重要元素更大/更孤立？

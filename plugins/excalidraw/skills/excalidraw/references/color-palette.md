# 色彩系統與品牌風格

**此檔案是所有色彩和品牌風格的唯一真相來源。** 若要自訂圖表的品牌風格，請編輯此檔 — 技能中的其他內容皆為通用設計方法論。

---

## 形狀色彩（語意化）

色彩用於編碼資訊，而非裝飾。每個語意用途都有對應的填充（fill）/ 邊框（stroke）配色。

| 語意用途 | 填充 | 邊框 |
|----------|------|------|
| 主要 / 中性 | `#3b82f6` | `#1e3a5f` |
| 次要 | `#60a5fa` | `#1e3a5f` |
| 第三 | `#93c5fd` | `#1e3a5f` |
| 起始 / 觸發 | `#fed7aa` | `#c2410c` |
| 結束 / 成功 | `#a7f3d0` | `#047857` |
| 警告 / 重設 | `#fee2e2` | `#dc2626` |
| 決策 | `#fef3c7` | `#b45309` |
| AI / LLM | `#ddd6fe` | `#6d28d9` |
| 停用 / 無效 | `#dbeafe` | `#1e40af`（搭配虛線邊框） |
| 錯誤 | `#fecaca` | `#b91c1c` |

**原則**：深色邊框搭配淺色填充，以產生對比。

## 架構圖色彩

適用於基礎設施、微服務和網路架構圖：

| 語意用途 | 填充 | 邊框 | 使用場景 |
|----------|------|------|----------|
| 外部實體 | `#a5d8ff` | `#1e1e1e` | Client、Browser、外部角色 |
| 閘道 / 入口 | `#b2f2bb` | `#1e1e1e` | API Gateway、Load Balancer、ingress |
| 應用服務 | `#d0bfff` | `#1e1e1e` | 微服務、業務邏輯 |
| 基礎設施 | `#c3fae8` | `#1e1e1e` | PostgreSQL、Redis、Kafka 等 |
| 命名空間 / 邊界 | `#d3f9d8` | `#22c55e` | Mesh boundary、namespace（使用 opacity 30） |

### 架構圖箭頭色彩

| 箭頭顏色 | 使用場景 |
|-----------|----------|
| `#22c55e` | mTLS / 安全連線 |
| `#06b6d4` | 資料庫連線 |
| `#8b5cf6` | 服務間呼叫 |
| `#4a9eed` | 外部 / ingress 連線 |

## 文字色彩（層級）

在浮動文字上使用色彩來建立視覺層級，而非使用容器。

| 層級 | 色彩 | 使用場景 |
|------|------|----------|
| 標題 | `#1e40af` | 區段標題、主要標籤 |
| 副標題 | `#3b82f6` | 子標題、次要標籤 |
| 內文 / 細節 | `#64748b` | 描述、註解、metadata |
| 淺色填充上的文字 | `#374151` | 淺色形狀內的文字 |
| 深色填充上的文字 | `#ffffff` | 深色形狀內的文字 |
| 預設邊框色 | `#1e1e1e` | 預設文字與元素邊框 |
| 備註 / 註解 | `#666666` | 次要備註 |

## 證據工件色彩

用於技術圖表中的程式碼片段、資料範例和其他具體佐證。

| 工件類型 | 背景 | 文字色彩 |
|----------|------|----------|
| 程式碼片段 | `#1e293b` | 語法高亮色彩 |
| JSON / 資料範例 | `#1e293b` | `#22c55e`（綠色） |

---

## 預設邊框與線條色彩

| 元素 | 色彩 |
|------|------|
| 箭頭 | 使用來源元素語意用途對應的邊框色 |
| 結構線（分隔線、樹狀、時間軸） | 主要邊框 (`#1e3a5f`) 或 Slate (`#64748b`) |
| 標記圓點（fill + stroke） | 主要填充 (`#3b82f6`) |
